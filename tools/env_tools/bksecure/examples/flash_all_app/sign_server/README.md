# Sign Server

This directory is used to simulate the sign server, which is responsible for signing binary files with cryptographic keys.

## Input Files

The following files are required as input to the sign server:

### Required Input Files (Binary Files Only)

**Note**: Sign server only requires binary files. JSON configuration files are NOT needed as input. All configuration parameters are passed via command line arguments.

1. **primary_all.bin** (Required)
   - Source: Copied from `build/bk7236n/xip/package/primary_all.bin`
   - Description: The primary application binary file that needs to be signed
   - Used for: Generating `primary_all_signed.bin` and `ota_signed.bin`

2. **bl2.bin** (Conditional - only when `bl1_secureboot_en` is True)
   - Source: Copied from `build/bk7236n/xip/package/bl2.bin`
   - Description: Bootloader binary file
   - Used for: Generating `primary_manifest.bin` for BL1 secure boot

### Configuration Files (Optional)

The sign server may use the following CSV configuration files if they exist in the working directory:

1. **security.csv** (Optional)
   - Description: Security configuration file containing key file paths and security settings
   - Contains: 
     - `img_sign_pubkey`: Path to public key PEM file
     - `img_sign_privkey`: Path to private key PEM file
     - `new_img_sign_pubkey`: Path to new public key PEM file (for key replacement)
     - `new_img_sign_privkey`: Path to new private key PEM file (for key replacement)
     - `update_img_sign_key_en`: Enable/disable key replacement
   - Note: In `flash_all_app` example, keys are passed via command line parameters, so this file is optional. However, some underlying tools may check for this file.

2. **partitions.csv** (Optional)
   - Description: Partition table configuration file
   - Contains: Partition definitions including sizes and offsets
   - Note: Used by some tools to determine partition sizes for signing operations
   - Source: Typically copied from `build/bk7236n/xip/package/partitions.csv` to `pack_server/immutable/`

3. **ota.csv** (Optional)
   - Description: OTA configuration file
   - Contains: OTA strategy and related settings
   - Note: May be used by some signing tools to determine OTA-related parameters

### Key Files (via command line parameters)

- **Private Key PEM file** (`--privkey`): Private key for signing
- **Public Key PEM file** (`--pubkey`): Public key for signature verification
- **New Private Key PEM file** (`--new_privkey`, optional): New private key when `replace_key_en` is True
- **New Public Key PEM file** (`--new_pubkey`, optional): New public key when `replace_key_en` is True

**Note**: In the `flash_all_app` workflow, keys are primarily passed via command line parameters. CSV files are optional and may be used by underlying tools for additional configuration or validation.

### JSON Files (NOT Required)

**Important**: Sign server does NOT require any JSON files as input. The following JSON files are used by other servers, NOT sign server:

- **pack.json**: Used by pack server, not sign server
- **primary_manifest.json**: Used by pack server (when BL1 secure boot is enabled), not sign server
- **otp_efuse_config.json**: Generated by pack server, not used by sign server

All signing operations in sign server are performed using binary files and command line parameters only. No JSON configuration files are needed.

## Input Directory

### Source Directory

Input files are copied from the build output directory:
- **Build output path**: `build/bk7236n/xip/package/`
- Files are copied by the main `run.py` script before sign server processing starts

### Working Directory

Input files are copied to the `sign_server/` directory before processing. The sign server reads input files from this directory (current working directory).

### Input File Copy Process

When `--build` option is used, the main `run.py` script performs the following file copies (see `run.py` lines 125-128):

1. **primary_all.bin** (Always copied)
   - Source: `build/bk7236n/xip/package/primary_all.bin`
   - Destination: `sign_server/primary_all.bin`
   - Code: `copy_file(f'{idk_path}/build/bk7236n/xip/package/primary_all.bin', f'{sign_path}/')` (line 125)

2. **bl2.bin** (Only when `bl1_secureboot_en` is True)
   - Source: `build/bk7236n/xip/package/bl2.bin`
   - Destination: `sign_server/bl2.bin`
   - Code: `copy_file(f'{idk_path}/build/bk7236n/xip/package/bl2.bin', f'{sign_path}/')` (line 128, inside `if args.bl1_secureboot_en:` block)

## Output Files

The sign server generates the following output files:

### Primary Output Files

1. **primary_all_signed.bin**
   - Description: Signed version of `primary_all.bin`
   - Contains: Application binary with MCUBoot header, signature, and TLV data
   - Used by: Pack server for creating `all-app.bin`

2. **ota_signed.bin**
   - Description: Signed OTA binary file
   - Generation:
     - **XIP mode**: Signed directly from `primary_all.bin`
     - **OVERWRITE mode**: Compressed and then signed from `compress.bin`
   - Used by: Pack server for creating `ota.bin`

3. **primary_manifest.bin** (Conditional - only when `bl1_secureboot_en` is True)
   - Description: BL1 manifest file for secure boot
   - Contains: Manifest information for BL1 secure boot verification
   - Used by: Pack server when BL1 secure boot is enabled

4. **pack_cmd_list.txt**
   - Description: Text file containing all signing commands executed
   - Format: Human-readable list of commands with descriptions
   - Used for: Debugging and documentation

### Intermediate Files (may be generated)

5. **compress.bin** (Only in OVERWRITE mode)
   - Description: Compressed version of `primary_all_signed.bin`
   - Used for: Creating OTA binary in OVERWRITE mode
   - Note: This is an intermediate file, may be cleaned up after use

## Processing Flow

1. **BL1 Signing** (if `bl1_secureboot_en` is True):
   - Input: `bl2.bin`
   - Output: `primary_manifest.bin`

2. **Application Signing**:
   - Input: `primary_all.bin`
   - Output: `primary_all_signed.bin`
   - Uses either standard key or new key (if `replace_key_en` is True)

3. **OTA Signing**:
   - **XIP mode**: Signs `primary_all.bin` directly
   - **OVERWRITE mode**: 
     - First compresses `primary_all_signed.bin` to `compress.bin`
     - Then signs `compress.bin` to create `ota_signed.bin`

## Important Notes

### Slot Size Configuration

If the offset or size of `primary_cpu0_app` in `partitions.csv` is changed, please update the `--slot-size` parameter in the sign command accordingly:

1. Build the project
2. Check the sign command in `build/bk7236n/xip/package/install/pack_cmd_list.txt`
3. Update the `--slot-size` parameter in `sign.py` to match the partition size

### Key Replacement

When `replace_key_en` is True:
- The signing process uses `new_privkey` and `new_pubkey` instead of the standard keys
- A custom TLV (0xa0) is added containing the new public key
- This allows for key rotation in secure boot scenarios

### OTA Type Differences

- **XIP (eXecute In Place)**: OTA binary is signed directly, no compression
- **OVERWRITE**: Application is compressed before signing to reduce OTA package size

## Output Directory

### Working Directory

The sign server processes files in the `sign_server/` directory. All output files are generated directly in this directory.

### Output Directory Structure

```
sign_server/
├── primary_all_signed.bin      # Primary output: Signed application binary
├── ota_signed.bin              # Primary output: Signed OTA binary
├── primary_manifest.bin        # Primary output (if bl1_secureboot_en): BL1 manifest
├── pack_cmd_list.txt           # Primary output: Command execution log
├── compress.bin                # Intermediate file (OVERWRITE mode only)
├── temp_before_compress        # Temporary file (OVERWRITE mode only)
├── temp_after_compress         # Temporary file (OVERWRITE mode only)
├── primary_all.bin             # Input file (copied from build output)
└── bl2.bin                     # Input file (if bl1_secureboot_en, copied from build output)
```

### File Transfer to Pack Server

After sign server processing completes, the main `run.py` script copies output files from `sign_server/` to `pack_server/_tmp/`:

- `primary_all_signed.bin` → `pack_server/_tmp/primary_all_signed.bin`
- `ota_signed.bin` → `pack_server/_tmp/ota_signed.bin`
- `pack_cmd_list.txt` → `pack_server/_tmp/pack_cmd_list.txt` (moved, not copied)
- `primary_manifest.bin` → `pack_server/_tmp/primary_manifest.bin` (if `bl1_secureboot_en` is True)

Intermediate files like `compress.bin` and temporary files are not copied to pack server.
