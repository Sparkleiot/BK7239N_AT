# Pack Server

This directory is used to simulate the pack server, which is responsible for packing signed binaries into final firmware images and generating OTP/eFuse configuration files.

## Input Files

The pack server receives input files from two sources:

### 1. Files from Sign Server (copied to `pack_server/_tmp/`)

These files are generated by the sign server and copied to the pack server's working directory:

1. **primary_all_signed.bin** (Required)
   - Source: Generated by sign server
   - Description: Signed primary application binary
   - Used for: Creating `app_signed.bin` and final `all-app.bin`

2. **ota_signed.bin** (Required)
   - Source: Generated by sign server
   - Description: Signed OTA binary file
   - Used for: Creating final `ota.bin`

3. **primary_manifest.bin** (Conditional - only when `bl1_secureboot_en` is True)
   - Source: Generated by sign server
   - Description: BL1 manifest file for secure boot
   - Used for: BL1 secure boot verification

4. **pack_cmd_list.txt** (Required)
   - Source: Generated by sign server
   - Description: Text file containing signing commands executed by sign server
   - Used for: Documentation and debugging

### 2. Files from Immutable Directory (copied to `pack_server/_tmp/`)

These files are configuration and binary files copied from `pack_server/immutable/`:

#### Binary Files

1. **bl2.bin** (Required)
   - Source: `pack_server/immutable/bl2.bin`
   - Description: Bootloader binary file
   - Used for: Creating `bootloader.bin` and `all-app.bin`

2. **partition.bin** (Required)
   - Source: `pack_server/immutable/partition.bin`
   - Description: Partition table binary file
   - Used for: Creating `all-app.bin`

3. **xip_a.bin** (Required)
   - Source: `pack_server/immutable/xip_a.bin`
   - Description: XIP application binary (original, before signing)
   - Used for: Creating `app_signed.bin` via BL2_SIGN action

4. **user_mfr.bin** (Required)
   - Source: `pack_server/immutable/user_mfr.bin`
   - Description: User manufacturing data binary
   - Used for: Creating `user_mfr_aes.bin` (encrypted version)

5. **provision.bin** (Conditional - only when `bl1_secureboot_en` is True)
   - Source: `pack_server/immutable/provision.bin`
   - Description: Provisioning data binary for BL1 secure boot
   - Used for: BL1 secure boot provisioning

#### Configuration Files (CSV)

6. **partitions.csv** (Required)
   - Source: `pack_server/immutable/partitions.csv`
   - Description: Partition table configuration file
   - Contains: Partition definitions including names, types, sizes, and offsets
   - Used for: Partition layout and binary placement

7. **bin.csv** (Required)
   - Source: `pack_server/immutable/bin.csv`
   - Description: Binary file configuration
   - Contains: Binary file definitions and metadata
   - Used for: Binary file processing and packing

8. **nvs.csv** (Required)
   - Source: `pack_server/immutable/nvs.csv`
   - Description: NVS (Non-Volatile Storage) configuration file
   - Contains: NVS key-value pair definitions
   - Used for: Generating `nvs.bin` and `nvs_key.bin`

9. **security.csv** (Required)
   - Source: `pack_server/immutable/security.csv`
   - Description: Security configuration file
   - Contains: Security settings including flash AES type, CRC enable, key file paths
   - Used for: Security-related packing operations

10. **ota.csv** (Required)
    - Source: `pack_server/immutable/ota.csv`
    - Description: OTA configuration file
    - Contains: OTA strategy and related settings
    - Used for: OTA binary generation

#### JSON Configuration Files

11. **pack.json** (Required)
    - Source: `pack_server/immutable/pack.json`
    - Description: Packing configuration file defining packing actions
    - Contains: Binary file definitions and packing actions (e.g., PACK_BL1_DOWNLOAD_BIN, BL2_SIGN, PACK_OTA_BIN)
    - Used for: Defining how binaries are packed into final firmware images

12. **primary_manifest.json** (Conditional - only when `bl1_secureboot_en` is True)
    - Source: `pack_server/immutable/primary_manifest.json`
    - Description: BL1 manifest JSON configuration
    - Used for: BL1 secure boot manifest generation

## Output Files

The pack server generates the following output files in the `pack_server/_tmp/install/` directory:

### Primary Output Files

1. **all-app.bin** (Required)
   - Description: Complete firmware image containing all components
   - Contains: bl2.bin, partition.bin, app_signed.bin, nvs.bin, nvs_key_pack.bin, user_mfr_aes.bin
   - Format: BL1 download format with pack headers
   - Used for: Flashing to device

2. **ota.bin** (Required)
   - Description: OTA update binary file
   - Contains: Signed application binary in OTA format
   - Format: OTA package format (XIP or OVERWRITE depending on configuration)
   - Used for: Over-the-air firmware updates

3. **bootloader.bin** (Required)
   - Description: Bootloader binary in BL1 download format
   - Contains: bl2.bin with pack header
   - Format: BL1 download format
   - Used for: Bootloader flashing

4. **otp_efuse_config.json** (Required)
   - Description: OTP/eFuse configuration file
   - Contains: Flash AES key, public key hash, and other OTP/eFuse settings
   - Format: JSON configuration file
   - Used for: Configuring device OTP/eFuse during manufacturing

5. **pack_cmd_list.txt** (Required)
   - Description: Text file containing all packing commands executed
   - Contains: Commands from both sign server and pack server
   - Format: Human-readable list of commands with descriptions
   - Used for: Debugging and documentation

### Additional Output Files (may be generated)

6. **bootloader_B.bin** (Optional)
   - Description: Backup bootloader binary (if bootloader OTA is enabled)
   - Used for: Bootloader OTA updates

7. **bootloader_ota.bin** (Optional)
   - Description: Bootloader OTA binary (if bootloader OTA is enabled)
   - Used for: Bootloader OTA updates

8. **provision_pack.bin** (Conditional - only when `bl1_secureboot_en` is True)
   - Description: Packed provisioning binary for BL1 secure boot
   - Used for: BL1 secure boot provisioning

## Output Directory Structure

The pack server generates output files in the following directory structure:

```
pack_server/
├── _tmp/                    # Working directory (temporary)
│   ├── install/             # Output directory
│   │   ├── all-app.bin
│   │   ├── ota.bin
│   │   ├── bootloader.bin
│   │   ├── otp_efuse_config.json
│   │   └── pack_cmd_list.txt
│   └── [intermediate files]  # Temporary files during processing
└── immutable/                # Input configuration files
    ├── bl2.bin
    ├── partition.bin
    ├── xip_a.bin
    ├── user_mfr.bin
    ├── partitions.csv
    ├── bin.csv
    ├── nvs.csv
    ├── security.csv
    ├── ota.csv
    └── pack.json
```

### Final Output Location

After pack server processing completes, output files are copied from `pack_server/_tmp/install/` to the final `install/` directory at the root of the example:

```
install/                      # Final output directory
├── all-app.bin
├── ota.bin
├── bootloader.bin
├── otp_efuse_config.json
└── pack_cmd_list.txt
```

## Processing Flow

1. **Prepare Working Directory**:
   - Create `pack_server/_tmp/` directory
   - Copy signed binaries from sign server
   - Copy configuration files from `immutable/` directory

2. **Generate OTP/eFuse Configuration**:
   - Input: Flash AES type, flash AES key, public key
   - Output: `otp_efuse_config.json`

3. **Process Packing Actions** (according to `pack.json`):
   - **BL2_SIGN**: Process `xip_a.bin` to create `app_signed.bin` (uses signed file from sign server)
   - **USER_MFR_AES**: Encrypt `user_mfr.bin` with NVS key to create `user_mfr_aes.bin`
   - **PACK_BL1_DOWNLOAD_BIN**: Pack binaries into BL1 download format
     - `bootloader.bin`: Pack `bl2.bin`
     - `all-app.bin`: Pack multiple binaries (bl2, partition, app_signed, nvs, nvs_key_pack, user_mfr_aes)
   - **PACK_OTA_BIN**: Create OTA binary from `app_signed.bin`

4. **Install Output Files**:
   - Copy generated files to `pack_server/_tmp/install/` directory
   - Files are then copied to final `install/` directory by the main script

## Important Notes

### File Dependencies

- `app_signed.bin` is created from `xip_a.bin` using the signed version from sign server (`primary_all_signed.bin`)
- `nvs.bin` and `nvs_key.bin` are generated from `nvs.csv` during packing
- `user_mfr_aes.bin` is created by encrypting `user_mfr.bin` with `nvs_key.bin`

### Working Directory

- Pack server processes files in `pack_server/_tmp/` directory
- All input files must be present in this directory before processing
- Output files are generated in `pack_server/_tmp/install/` subdirectory

### Configuration Files

- All CSV and JSON configuration files must be present in `pack_server/immutable/` before processing
- These files are copied to `pack_server/_tmp/` at the start of processing
- Changes to configuration files require updating files in `immutable/` directory
